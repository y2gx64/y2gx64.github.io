## Introduction
I will be explaining how i did ret2win.c found at <https://github.com/Crypto-Cat/CTF/blob/main/pwn/binary_exploitation_101/03-return_to_win/ret2win.c>

## Step one - inspect source code

```
#include <stdio.h>␍
␍
void hacked()␍
{␍
    printf("This function is TOP SECRET! How did you get in here?! :O\n");␍
}␍
␍
void register_name()␍
{␍
    char buffer[16];␍

I can infer that the my input will be placed into the variable `buffer` with a size of 16 bytes. So i wanted to test if i can achieve buffer overflow here. 

## Step two - buffer overflow?

When prompted to enter a name, i went with something with more than 16 characters, such as 32 characters for example. 

![](assets/posts/PWN/phoenix_format_three/output.png)

The image above shows that a buffer overflow is possible. Even better, it shows that the EIP attempted to execute instructions at the address `0x41414141` which was my input. This means that i need to overflow a specific value to "trick" EIP into executing instructions from it. 


## Step three - how much to overflow?

To figure out how much to overflow, i disassembled the function __register_name__ which gave me the following output.

![](assets/posts/PWN/phoenix_format_three/helpful.png)

Based on the disassembly above, i am supposed to overflow and overwrite EBP , causing the program to return to the wrong address. To retrieve the value of EBP, I set a breakpoint at `0x08049202` before __ret__ is called.

![](assets/posts/PWN/phoenix_format_three/bp1.png)

At the breakpoint, the following was shown.

![](assets/posts/PWN/phoenix_format_three/stack.png)

This shows that ebp is `0xffffd508`, minus 4 giving `0xffffd504`. This means that the 4 bytes after `0xfffffd504` is the return address which is what i want to replace via overflowing my values into it. I can fill up to `0xffffd504` with junk bytes (20) which will be letter "A" in this case and the value after that will be the memory address of __register_name__.  This gave me the following script to use:

```
import sys

changeme = b"A"*20
changeme += b"\x86\x91\x04\x08"

sys.stdout.buffer.write(changeme)

```
## Step four - running the payload
Generate the file with the payload:

```
python3 <script gen> > payload 
```
Use it as input for ret2win:

```
