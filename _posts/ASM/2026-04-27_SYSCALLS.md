## Syscalls
Syscalls are operations performed by the kernel of the operating system. Examples of such syscalls/operations are: 

1. sys_write
2. sys_read
3. sys_open 

Usually you dont have go through this many steps if you're using a language such as Python where its automatically handled for you and all you have to do is type `print()` 

## How it is done 
Firstly, you put the system call number into the `EAX` register. Secondly, store the arguments needed for the system call to its corresponding 
registers.

In x86, these registers usually are:
1. `EAX` to store the system call number
2. `EBX` for the first argument
3. `ECX` for the second argument
4. `EDX` for the third argument
5. `ESX` for the fourth argument
6. `EDI` for the fifth argument

In x64, these registers are:
1. `RAX` - to store the system call number.
2. `RDi` - for the first argument
3. `RSI` for the second argument
4. `RDX` for the third argument


Thirdly, call the relevant interrupt to notify the CPU that a system call needs to be run. In x86, this is `int 0x80` while in x64 this is `syscall`


## Example

### sys_write
This is the code to print 'hello' in x86:

```
segment .data 
  hellomsg db `hello\n`
  lenhellomsg equ $-hellomsg

segment .text 
  global _start

_start:
  mov	edx,lenhellomsg
  mov	ecx,hellomsg
  mov	ebx,1
  mov	eax,4
  int	0x80

  mov	eax,1
  int	0x80
```

Firstly, the length of the message is moved into the register `EDX`. Secondly, the actual message is moved into the register `ECX`. Thirdly, unsigned int `1` is moved into `EBX`. Fourthly as the `sys_write` value is 4, the value 4 is moved into `EAX`. Lastly, an interrupt is called.


This is the code to print 'hello' in x64:

```
segment .data 
  hellomsg db `hello\n`
  lenhellomsg equ $-hellomsg

segment .text 
  global _start

_start:
  mov rax,1 ;code
  mov rdi,1 ;unsigned int
  mov rsi, hellomsg ;const char *buf
  mov rdx,lenhellomsg ;size t_count
  syscall

  mov	rax,60
  syscall
```
