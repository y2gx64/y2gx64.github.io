## Overflow flag 
For example, in a 8 bit register, the range for an integer is `-128` to `127`. If the integer is any higher than that it will cause an overflow. For example:

```
mov al, 96  ;01100000
add al, 32  ;00100000
```
Despite the result being `128`, as it cannot be represented in this 8 bits, it overflows and becomes `-128` which is obviously wrong. Hence the overflow flag is set.

![](assets/posts/ASM/flags/OF_1.png)

Another example is:

```
mov al, -96
add al, -33
```
Despite the result being `-129`, as it cannot be represented in this 8 bits, it overflows and become `127` which is wrong. Hence the overflow flag is set to `1`.

![](assets/posts/ASM/flags/OF_2.png)

The opcode `add` is used instead as subtracting a negative number is the equivalent of addition which is not what we want here.

## Sign flag
For example, in a 8 bit register the most significant bit (MSB) is the one on the right. If it is set to `1`, sign flag will be `1` otherwise it remains being `0`. An example being

```
mov al, 12 ;00001100
add al, -13  ;11110011
```
This will cause the MSB to be `1` which causes the sign flag to be set to `1`.

![](assets/posts/ASM/flags/SF_1.png)

## Auxiliary carry flag
To cause the AF to be set, there needs to be carry from a bit 3 to a bit 4 in an arithmetic operation. For example:

```
mov al, 12  ;00001100
add al, 12  ;00001100
```
When you manually add them, there is a carry over from the third to the fourth bit. This causes the auxiliary flag to be set to `1` .

![](assets/posts/ASM/post12/AF_1.png)

## Parity flag
To cause the PF to be set, the number of `1` bits needed to be even. For example:

```
mov al, 12  ;00001100
add al, 12  ;00001100

```
The final result is `24` which is `00011000`. As there are `2` one bits, the parity flag to be set to `1`.

![](assets/posts/ASM/flags/PF_1.png)

## Carry flag
To cause the CF to be set, there must be a carry over from bit 7 to bit 8 (MSB), for example:

```
mov al, -96;  10100000
add al, -12; 11110100
```
When you manually add them,there is a carry over from the seventh bit to the eighth  bit, hence the carry flag is set to `1`.

![](assets/posts/ASM/flags/CF_1.png)

## Zero flag
To cause ZF to be set to `1`, the result from the previous arithmetic or comparison operation must be zero:

```
mov al, 2; 00000010
add al, -2; 11111110
```
Since the result from 2 + -2 is zero, the zero flag is set to `1`.

![](assets/posts/ASM/flags/ZF_1.png)
